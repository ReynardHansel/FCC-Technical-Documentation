Introduction

Cascading Style Sheets (CSS) describes how HTML elements are to be displayed on screen, paper, or in other media. It saves a lot of work as it can control layouts and designs of multiple web pages at once.

External stylesheets are stored in css files (.css)

Selectors

Now that you have created the place where you can write CSS (whether it's a style tag or a new .css file), it's time to write and use CSS.

CSS targets an HTML attribute, then tells it to display a certain way. So to use CSS, first you have to select/target the element you want to style. This is done by what's called a "Selector" in CSS

Ex:

                
    p {
        color: red;
        font-size 20px;
        font-style: italic;
    }

Now all the <p> tags in the html file will have red color, 20px big, and displayed in italic.

There's a lot of CSS selectors but the most common used ones are the class and id selectors. You do this by defining a class and id in the html tag (Ex: <div class="card" id="first-card"></div>). And then to target a class, put a . in front of the class name. To target an id, put a # in front of the id name.

Ex:

                
    .card {
        padding: 2em;
        margin: 2em auto;
        font-size: 1.3em;
    }

    #first-card {
        background-image: url (background.jpg)
    }
CSS Properties

CSS properties specifies what to style of the targeted HTML elements

For example, you can make a <div> background to be red, the text in that <div> to have the color white, and all the fonts in that page Open Sans. You can do that by doing something like this:

                
    * {
        font-family: Open Sans;
    }

    div {
        background-color: red;
    }

    div p {
        color: white;
    }

At the example above, 3 CSS properties are being used. font-family, background-color and color.

Specificity

If you have different styles that targets the same HTML element, CSS will run the code that is more specific/have higher specificity.

Ex:

                
    <html>
        <head>
            <style>
            #demo {color: blue;}
            .test {color: green;}
            p {color: red;}
            </style>
        </head>
        <body>
                        
        <p id="demo" class="test">HelloWorld!</p>
                        
        </body>
    </html>

There are 3 selectors that targets the same element and each of them has different properties. But the text will be shown in the color blue because an id(#) is more specific than a class and element selector.

Specificity Hierarchy

Below are the order of the most specific (most priority) to the least specific (less priority) that you'll see on a daily basis:

  1. Inline styles - Example: <h1 style="color: pink;"></h1>
  2. IDs - Example: #navbar
  3. Classes, pseudo-classes, attribute selectors - Example: .test, :hover, [href]
  4. Elements and pseudo-elements - Example: h1, :before
CSS Units

CSS has several different units for expressing a length.

Many CSS properties take "length" values, such as width, margin, padding, font-size, etc.

Length is a number followed by a length unit, such as 10px, 2em, etc.

CSS units could be divided to 2 types of units: Fixed units and Relative units

  • Fixed units

    • Fixed units have fixed sizes and will appear as exactly that size.

    Generally fixed units are not reccomended because it's unresponsive. It does not scale at all no matter what the viewer's screen size is or what's their font preference. It stays exactly the same and often look bad on different screen sizes.

    The most used fixed unit is px (pixels). There are others such as cm (centimeters), mm (milimeters), in (inches), etc.

Fonts

Fonts is a really important part of a website as it could make your site looks incredibly awesome, or make people dislike it. You can change the default HTML font in CSS by using the font-family: property.

Ex:

                
    p {
        font-family: sans-serif;
    }

Now all the <p> elements will be shown in sans-serif.

Flexbox & Grid

Flexbox & Grid are the most popular way to design web pages easily. By just typing display: flex; or display: grid; as the display property, you get access to use flexbox/grid and their respective properties.

The element that you give display: flex; or display: grid; will be the flex/grid container. The children/elements in the HTML below that will be called flex items and grid items.

There are several properties that can be used for the container and their items. For complete information and how to use flex and grid, you can head to www.w3schools.com and check their CSS FLexbox and CSS Grid section.

Animation

There are 2 ways you could animate your HTML using CSS:

  • Transition
  • Animation (@keyframes)

Transition

CSS transitions allows you to change property values smoothly, over a given duration. It is mostly used to transition to a hover state smoothly. These are the properties for transition:

  • transition-delay
  • transition-duration
  • transition-property
  • transition-timing-function

However the most important part is transition-duration, transition will not work if the duration isn't specified, because the default value is 0

Ex:

                
    div {
        width: 100px;
        background-color: red;
        transition: all;
        transition-duration: 1s;
        transition-delay: .5s;
    }

    div:hover {
        width: 200px;
        transform: translateX(50px);
        background-color: yellow;
    }

Animate (@keyframes)

An animation lets an element gradually change from one style to another. You can change as many CSS properties you want, as many times as you want. To use CSS animation, you must first Keyframes hold what styles the element will have at certain times.

First, we add the properties animation name & animation-duration to the elements we target. (Note: if these 2 properties are not specified, then no animations will occur.

Next, we use @keyframes and then we specify what would happen within the animation. Animation will start from 0% and end in 100%

Ex:

                
    div {
        width: 100px;
        background-color: red;
        animation-name: animate;
        animation-duration: 4s;
    }

    @keyframes animate {
        25% {
            width: 200px;
        }
        50% {
            background-color: green;
        }
        75% {
            translateX (200px);
        }
        100% {
            border-radius: 50%;
        }
    }

After 1s, the div will grow to 200px. After 2s, the background-color will now be green. After 3s, the div will move to the right by 200px. After 4s, the div will become a circle. And then the animation ends and will go back to it's original postition.

If you want the animation to stay in it's last place, you could type animation-fill-mode: forwards;. For detailed explanations about animation, you could go to w3schools animation section here

Media Query

Sometimes we made a pretty good job making the frotend design of a web. But then you reduce the width of the browser or open it in your phone, it breaks apart and looked terrible. The layout got broken and the elements are everywhere. Sometimes things are too big or not centered enough, etc

That's where Media Query comes in.

Media Query lets you customize the css that works at a certain screen size.

To use it, type @media and then specify when should the code within the @media runs. Add {} and type the CSS inside it.

Ex:

                
    @media (min-width: 600px) {
        div {
            background-image: mobile-view-image.jpg;
            background-size: cover;
            font-size: .7rem;
        }
    }

Now when the screen gets 600px and wider, the background in the <div> and the font size will change.

Responsive CSS

Responsive CSS design means that no matter what screen size the viewer has, the design will still look good and makes the viewer still comfortable staying & interacting with it.

Some things to keep in mind when writing CSS and make it responsive:

  • Avoid using fixed units like pixels.
  • Design for mobile first, then use media query to fix the look in desktop view.
  • Use modern design techniques
  • Avoid fixed dimensions like width and height and try using min-height and max-width instead (note: width and height are not bad and are still used, just try sometimes to break out of it and use min-height & max-width instead)
